# PELT 변동점 감지 — 시계열이 "언제" 바뀌었는지 찾기

> **이 분석이 하는 일**: "그래프를 눈으로 보면 여기서 바뀐 것 같은데" → 이걸 통계로 증명하기
>
> **프로젝트에서 쓰는 곳**: Phase 2 — 월별 로판 신작 수 시계열에서 "재혼황후 이후 진짜로 구조가 바뀌었나?" 탐지

**원논문(Tier 1)**: Killick, R., Fearnhead, P., Eckley, I.A. (2012). *Optimal Detection of Changepoints With a Linear Computational Cost.* Journal of the American Statistical Association, 107(500), 1590–1598.

**라이브러리 논문(Tier 1)**: Truong, C., Oudre, L., Vayatis, N. (2020). *Selective review of offline change point detection methods.* Signal Processing, 167, 107299. arXiv:1801.00826

---

## 쉬운 비유로 먼저 이해하기

반 학생들의 월별 도서관 대출 건수를 추적하고 있다고 해봐요.

```
대출 건수
│              ┌──────────
│              │
│──────────────┘
└─────────────────────────→ 시간
              ↑
         어느 날 갑자기 학교 도서관을 리모델링 → 이용자 폭증
```

눈으로 보면 "여기서 바뀐 것 같다"는데, **통계적으로 "이 시점이 진짜 변동점"이라고 어떻게 증명하냐?** → PELT가 하는 일이에요.

PELT는 "여기서 쪼갰을 때 각 구간 내부가 가장 균질해지는 지점"을 찾아요.

---

## 핵심 아이디어: "비용 + 패널티" 최소화

PELT가 푸는 문제를 말로 설명하면:

> "시계열을 여러 구간으로 나눌 수 있는데, 구간을 많이 나눌수록 각 구간 내부가 균질해진다. 하지만 구간을 너무 많이 나누면 노이즈도 변동점으로 잡아버린다. 이 둘의 균형을 찾아라."

수식으로는:

$$\min \left[ \sum \text{(각 구간의 분산)} + \beta \times \text{(변동점 개수)} \right]$$

- **구간 분산**: 구간 내 데이터가 균질할수록 낮음
- **β (패널티)**: 변동점 1개 추가할 때마다 내야 하는 "비용". 높을수록 변동점을 더 확실할 때만 인정

**비유**: 레스토랑에서 코스 요리를 추가할 때마다 1만원을 내야 한다고 하면, 배고프지 않으면 추가 안 하겠죠. β가 높을수록 "확실히 다른 구간"일 때만 변동점을 인정하는 거예요.

---

## β(패널티) 설정이 가장 중요

```python
import ruptures as rpt

signal = monthly_new_works.values.reshape(-1, 1)  # numpy 배열로 변환

algo = rpt.Pelt(model="rbf").fit(signal)

# β를 바꿔가며 결과 비교해보기
for pen in [1, 5, 10, 20, 50]:
    result = algo.predict(pen=pen)
    print(f"β={pen}: 변동점 {len(result)-1}개 → {result[:-1]}")
```

**β가 너무 낮으면**: 노이즈를 변동점으로 잡음 (과탐지)
```
β=1: 변동점 23개 → [5, 8, 12, 15, ...]  # 매달 바뀐다고???
```

**β가 너무 높으면**: 진짜 변화도 놓침 (미탐지)
```
β=50: 변동점 0개 → []  # 아무것도 안 잡힘
```

**적절한 β 찾는 방법**:
- Elbow method: β를 1부터 50까지 바꾸면서 변동점 수 그래프 → 꺾이는 지점이 적절한 값
- BIC 기준: `pen = log(데이터 수)` (많이 쓰는 방법)

## 비용 함수(model) 선택

```python
rpt.Pelt(model="rbf")   # 평균 + 분산이 동시에 바뀌는 경우 (가장 범용적)
rpt.Pelt(model="l2")    # 평균만 바뀌는 경우 (로판 신작 수처럼 카운트 데이터)
rpt.Pelt(model="l1")    # 이상치가 많아도 강건하게
```

로판 신작 수는 단순 카운트라서 `"l2"` 또는 `"rbf"` 둘 다 시도해보고 더 의미 있는 결과를 선택해요.

## 핵심 가정 — "이게 맞아야 PELT를 쓸 수 있어요"

| 가정 | 쉬운 말 | 깨지면? |
|------|--------|--------|
| 변화가 갑작스럽게(abrupt) 일어난다 | 어느 달부터 갑자기 달라진다 | 수년에 걸쳐 서서히 변하면 감지 못함 |
| 각 구간 내부는 통계적으로 비슷하다 | 같은 구간 안에서도 트렌드가 있으면 안 됨 | STL로 트렌드 제거 후 PELT 적용 권장 |
| β가 적절하게 설정돼 있다 | 패널티가 너무 크거나 작으면 결과 왜곡 | Elbow method로 최적값 탐색 필요 |

---

## 이럴 때 PELT가 실패해요

### ❌ 실패 1: 서서히 바뀌는 변화는 못 찾아요
```
신작 수: 10→11→12→13→14→15... (매달 1개씩 증가)
```
이런 점진적 증가는 PELT가 "변동점"으로 잡지 않아요. STL 트렌드가 더 적합.

### ❌ 실패 2: 변동점 전후에 데이터가 너무 적으면
- 변동점 탐지 후 각 구간에 데이터가 최소 5~10개는 있어야 해요
- 월별 데이터면 변동점 전후 각각 6개월 이상 필요

### ❌ 실패 3: "변동점 = 이벤트 원인"으로 단정하면 안 돼요
- PELT가 2019년 10월을 변동점으로 찾았다고 해서 "재혼황후가 원인"이라고 말할 수 없어요
- **변동점 탐지는 "언제 바뀌었나"만 알려줌**
- "왜 바뀌었나"는 다음 단계(ITS)에서 검정해요

## 로판 프로젝트에서 교차검증하기

```python
# PELT가 찾은 변동점 날짜 확인
changepoints = result[:-1]  # 마지막 값은 시계열 끝 인덱스이므로 제외
changepoint_dates = [df.index[cp] for cp in changepoints]
print("탐지된 변동점:", changepoint_dates)

# 알고 있는 이벤트 날짜와 비교
known_events = {
    'E001_재혼황후_웹툰': '2019-10-01',
    'E003_선재업고튀어_드라마': '2024-04-08',
}

# 변동점이 이벤트 ±3개월 안에 들어오면 관련성 높음
```

**핵심 질문**: PELT가 찾은 시점이 알고 있는 이벤트 날짜 근처인가?
- 맞으면 → "이 이벤트가 구조적 변화와 관련 있다는 증거"
- 안 맞으면 → "다른 외부 요인이 있을 수 있다" → 추가 조사

## 면접에서 "왜 변동점 감지를 썼나요?" 질문이 나오면

> "단순히 그래프를 눈으로 보고 '여기서 바뀐 것 같다'고 말하는 건 주관적이에요. PELT는 시계열에서 통계적으로 가장 설명력 있는 분할 지점을 찾아주는 알고리즘입니다. 세그먼트 내부 분산의 합을 최소화하면서 변동점 수에 패널티를 부과하는 최적화 문제를 푸는 방식이고, 탐지된 변동점이 재혼황후 웹툰 연재 시점(2019년 10월)과 일치하는지 교차검증했습니다. 이 교차검증이 맞으면 다음 단계에서 ITS로 실제 인과 효과를 검정합니다."
