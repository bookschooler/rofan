# STL 분해 — 트렌드·계절성·잔차 분리하기

> **이 분석이 하는 일**: "로판 신작이 진짜 늘고 있는 건지, 아니면 연말마다 원래 신작이 많아서 그런 건지" 구분하기
>
> **프로젝트에서 쓰는 곳**: Phase 2 — 월별 로판 신작 수를 세 가지 성분으로 분리

**원논문(Tier 1)**: Cleveland, R.B. 외 (1990). *STL: A Seasonal-Trend Decomposition Procedure Based on Loess.* Journal of Official Statistics, 6(1), 3–73.

---

## 쉬운 비유로 먼저 이해하기

카페 매출을 월별로 그래프로 그렸다고 생각해봐요.

```
매출
│       ╭─╮   ╭─╮   ╭─╮
│   ╭─╮ │ │ ╭─╯ ╰─╮ │ │
│───╯ ╰─╯ ╰─╯     ╰─╯ ╰──
└─────────────────────────→ 시간
  12월마다 크리스마스 특수로 급등
```

이걸 보고 "카페가 성장하고 있나?"를 판단하기 어려워요. 12월만 돌아오면 매출이 올라가니까요.

STL은 이 데이터를 이렇게 쪼개줘요:

```
원본 = 트렌드 + 계절성 + 잔차
        (성장세)  (12월 특수)  (나머지)
```

로판으로 바꾸면:
- **트렌드(T)**: "로판 신작이 연도별로 꾸준히 늘고 있다"
- **계절성(S)**: "연초마다 신작이 많다 (연말에 완결된 작품이 1월에 새로 시작)"
- **잔차(R)**: "2019년 10월에 갑자기 이상하게 많다" (재혼황후 효과 가능성)

---

## 수식 (겁먹지 마세요, 딱 1개예요)

$$Y_t = T_t + S_t + R_t$$

읽는 법: **"t월의 신작 수 = 트렌드 + 계절성 + 잔차"**

이게 가법(덧셈) 분해예요. 세 개를 더하면 원본이 돼요.

> **언제 곱셈으로 바꾸나?** 트렌드가 커질수록 계절성의 들쭉날쭉 폭도 함께 커지면 곱셈 사용. 예: 매출이 10배 늘어나면서 12월 특수도 10배 커짐 → 곱셈. 들쭉날쭉 폭이 일정하면 덧셈.

---

## LOESS가 뭔데요?

STL 이름에 "LOESS"가 들어가는 이유: 트렌드를 추정할 때 LOESS 방식을 쓰기 때문이에요.

**LOESS(로컬 가중 회귀)**: "지금 이 시점의 트렌드를 추정할 때, 먼 옛날 데이터보다 최근 데이터를 더 많이 참고한다"

**비유**: 여러분이 "요즘 로판이 인기 있나?"를 판단할 때, 2013년 데이터보다 2023년 데이터를 더 많이 참고하는 것과 같아요.

---

## 코드로 보기

```python
from statsmodels.tsa.seasonal import STL

result = STL(
    monthly_counts,   # 월별 로판 신작 수 시계열
    period=12,        # 1년 = 12개월 주기
    robust=True       # 이상치(갑자기 튀는 달) 있어도 망가지지 않도록
).fit()

result.trend    # 트렌드 성분만 꺼내기
result.seasonal # 계절성 성분만 꺼내기
result.resid    # 잔차 성분만 꺼내기

result.plot()   # 4개 그래프 한 번에 보기
```

## 핵심 가정 — "이게 맞아야 STL을 쓸 수 있어요"

| 가정 | 쉬운 말 | 깨지면? |
|------|--------|--------|
| 계절 패턴이 해마다 비슷하다 | 매년 1월에 신작이 많다는 패턴이 일정하다 | 어떤 해는 1월, 어떤 해는 3월에 많으면 계절성 추정이 엉킴 |
| 데이터가 최소 2~3주기 이상 있다 | 12개월 주기라면 최소 24~36개월 데이터 | 데이터가 너무 짧으면 계절성 추정 불가 |
| 중간에 갑자기 패턴이 완전히 바뀌지 않는다 | 2019년 전과 후가 완전히 다른 세계가 아니다 | 갑자기 바뀐 경우 PELT로 먼저 찾고 구간 분리 필요 |

## 이럴 때 STL이 실패해요

### ❌ 실패 1: period를 잘못 넣었을 때
```python
# 월별 데이터인데 주간 주기로 잘못 설정
STL(monthly_data, period=7)  # 7은 주간 데이터용!
```
→ 계절성과 트렌드가 완전히 뒤엉킴. **period는 데이터 주기(월별이면 12)와 반드시 맞춰야 함**

### ❌ 실패 2: 데이터가 너무 짧을 때
- 2022년~2023년 2년 데이터로 연간 계절성 추정 → 겨우 2바퀴밖에 없어서 불안정
- **경험 법칙: 최소 3년 이상 권장**

### ❌ 실패 3: 잔차가 너무 클 때
- 잔차(R) 그래프에서 패턴이 보이면 STL이 설명 못 한 무언가가 있다는 신호
- 예: 잔차에 2019년 10월만 유독 크면 → 재혼황후 이벤트 효과가 트렌드/계절성으로 설명이 안 되는 것
- → 다음 분석(PELT, ITS)으로 넘어가기

---

## 로판 프로젝트에서 구체적으로 어떻게 쓰나요?

**분석 흐름**:
1. 2013년~현재까지 월별 로판 신작 수 데이터 준비
2. STL로 트렌드 분리 → 트렌드 기울기가 양수면 "로판은 성장 중"이라고 말할 수 있음
3. 잔차를 보고 "이상하게 튀는 달"을 확인 → 이 시점이 이벤트와 맞나?
4. 이 이상치 시점을 PELT에 넘겨서 "언제 구조적으로 변했나" 탐지

---

## 면접에서 "왜 STL 썼나요?" 질문이 나오면

> "월별 로판 신작 수에는 연초마다 신작이 집중되는 계절성이 섞여 있어요. 그걸 분리하지 않고 트렌드를 보면 '원래 1월마다 많던 거'를 '성장하고 있다'고 잘못 읽을 수 있어요. STL로 트렌드·계절성·잔차를 분리해서, 계절 패턴을 제거한 순수한 성장 방향을 봤습니다. 잔차에서 2019년 10월에 이상치가 확인됐고, 다음 단계에서 PELT로 그 시점이 통계적 변동점인지 확인했습니다."
