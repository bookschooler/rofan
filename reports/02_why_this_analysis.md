## Step 2-1: 시계열 집계

### 무엇을 하나요?
`all_works_integrated.csv`(16,337행)에서 **월별 로판 신작 수** 시계열을 만듭니다.
"2013년 1월에 몇 편, 2월에 몇 편…" 식으로 월 단위로 묶어주는 작업입니다.

### 왜 필요한가요?
STL 분해와 PELT 변동점 탐지는 모두 **등간격 시계열**을 입력으로 받습니다.
개별 작품 행(16,337개)을 그대로 넣으면 안 되고, 먼저 월별로 집계해줘야 합니다.

### 어떻게 하나요?
1. `start_date`를 `'YYYY-MM'` 단위 Period로 변환
2. `groupby("month").size()` → 월별 신작 수
3. 결측 월 0 채우기 — 신작이 0편인 달도 시계열에 포함해야 STL이 정확함
4. `data/processed/02_timeseries_monthly.csv`로 저장

### 분석 대상 필터
- `content_form == 'serialized'`만 사용 (book_edition 13,380건, spinoff 80건 제외)
- `all_works_integrated.csv`는 이미 로판 전용 수집 결과이므로 장르 필터 불필요

## Step 2-1 (플랫폼별): 플랫폼별 시계열 집계

### 무엇을 하나요?
전체 시계열과 동일한 집계를 **카카오페이지 / 네이버 시리즈 / 네이버 웹툰** 3개 플랫폼으로 나눠서 합니다.

### 왜 필요한가요?
전체 합산만 보면 플랫폼 간 성장 속도 차이, 역전 시점, 플랫폼별 계절성 차이를 놓칩니다.
나중에 Phase 3 ITS 분석에서 이벤트 효과를 플랫폼별로 검증할 때 기반 데이터가 됩니다.

### 어떻게 하나요?
- `groupby(["platform", "month"]).size()` → `unstack(platform)` 으로 피벗
- 없는 달은 0으로 채워 각 플랫폼의 시계열 길이를 동일하게 맞춤
- 누적 면적 차트(상단) + 개별 꺾은선(하단) 2패널로 시각화

## Step 2-2: STL 분해

### 무엇을 하나요?
월별 신작 수 시계열을 **3가지 성분**으로 분리합니다.

```
원본 시계열 = 추세(Trend) + 계절성(Seasonal) + 잔차(Residual)
```

- **추세(T):** 로판 시장이 장기적으로 성장/정체/하락 중인가
- **계절성(S):** 특정 달에 신작이 몰리는 패턴이 있는가 (예: 방학 시즌)
- **잔차(R):** 추세·계절성으로 설명 안 되는 이상치 — 외부 이벤트 영향을 여기서 잡아냄

### 왜 필요한가요?
신작 수 원본만 보면 "2024년에 왜 줄었지?" 같은 질문에 답하기 어렵습니다.
STL로 분리하면 "추세는 아직 상승인데, 계절성 효과가 없는 달이라 낮아 보이는 것" 같은 구분이 가능해집니다.
Phase 3 ITS 분석에서 이벤트 효과를 측정할 때 계절성·추세를 통제하기 위한 기반이기도 합니다.

### 어떻게 하나요?
- **균등 간격 보정 (전처리):** 2013년 초반 8개월 결측 구간을 `pd.date_range`로 리인덱싱 후 선형 보간. STL은 내부적으로 등간격 시계열을 가정하기 때문에 빠진 달이 있으면 계절성 추정이 왜곡됨.
- `STL(period=12, robust=True)` 실행 — 12개월 주기, 이상치에 강건한 옵션
- 추세에 선형 회귀 → 기울기(편/월), 연간 증가율, p-value
- **Mann-Kendall 비모수 검정 병행** → 선형 가정 없이 단조 추세 유무 확인 (S자 성장곡선에도 유효)
- 계절성 월별 평균 → 성수기/비수기 달 확인
- 잔차 상위/하위 5개 월 → 이상 시점 식별

## Step 2-3: PELT 변동점 감지

### 무엇을 하나요?
시계열에서 **구조가 바뀐 시점(변동점, changepoint)**을 자동으로 찾아냅니다.
예: "2018년 이전까지는 월 20~30편 수준이었는데, 이후 갑자기 80편 이상으로 레벨이 올라갔다" 같은 전환점을 탐지합니다.

### 왜 필요한가요?
Phase 3 ITS 분석에서 "이 이벤트가 시장에 영향을 줬다"를 검증하려면, 먼저 **어디서 구조가 바뀌었는지**를 파악해야 합니다.
PELT가 찾아낸 변동점과 알려진 이벤트(재혼황후 웹툰화 2019-10, 선재업고튀어 드라마화 2024-04)의 시기를 비교해 인과 분석 대상을 좁힙니다.

### 어떻게 하나요?
- STL의 잔차를 제거한 `trend + seasonal` 신호를 입력으로 사용 (순수 구조 변화만 탐지)
- `ruptures.Pelt(model='rbf')` 실행
- **Elbow Method 자동화:** penalty 1~99 전 범위 탐색 → 변동점이 3~6개 나오는 구간의 최대 penalty 자동 선택 (과다/과소 탐지 모두 방지하는 가장 보수적 기준)
- 탐지된 변동점 날짜 vs 알려진 이벤트 날짜 거리 비교
- 변동점 전후 구간 월 평균 신작 수 비교로 변화 크기 정량화
- **Welch's t-test로 전후 평균 차이 통계 검증** → "정말 달라진 건지" p-value로 확인 (등분산 가정 없는 버전 사용)
